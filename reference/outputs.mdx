title: Outputs
description: All output values for the terraform-aws-eks root module, grouped by category.
---

## Cluster

<ResponseField name="cluster_arn" type="string">
  The Amazon Resource Name (ARN) of the cluster.
</ResponseField>

<ResponseField name="cluster_certificate_authority_data" type="string">
  Base64 encoded certificate data required to communicate with the cluster.
</ResponseField>

<ResponseField name="cluster_endpoint" type="string">
  Endpoint for your Kubernetes API server.
</ResponseField>

<ResponseField name="cluster_id" type="string">
  The ID of the EKS cluster. Note: currently a value is returned only for local EKS clusters created on Outposts.
</ResponseField>

<ResponseField name="cluster_name" type="string">
  The name of the EKS cluster.
</ResponseField>

<ResponseField name="cluster_oidc_issuer_url" type="string">
  The URL on the EKS cluster for the OpenID Connect identity provider.
</ResponseField>

<ResponseField name="cluster_dualstack_oidc_issuer_url" type="string">
  Dual-stack compatible URL on the EKS cluster for the OpenID Connect identity provider.
</ResponseField>

<ResponseField name="cluster_version" type="string">
  The Kubernetes version for the cluster.
</ResponseField>

<ResponseField name="cluster_platform_version" type="string">
  Platform version for the cluster.
</ResponseField>

<ResponseField name="cluster_status" type="string">
  Status of the EKS cluster. One of `CREATING`, `ACTIVE`, `DELETING`, `FAILED`.
</ResponseField>

<ResponseField name="cluster_primary_security_group_id" type="string">
  Cluster security group that was created by Amazon EKS for the cluster. Managed node groups use this security group for control-plane-to-data-plane communication. Referred to as 'Cluster security group' in the EKS console.
</ResponseField>

<ResponseField name="cluster_service_cidr" type="string">
  The CIDR block where Kubernetes pod and service IP addresses are assigned from.
</ResponseField>

<ResponseField name="cluster_ip_family" type="string">
  The IP family used by the cluster (e.g. `ipv4` or `ipv6`).
</ResponseField>

<ResponseField name="cluster_control_plane_scaling_tier" type="string">
  The EKS Provisioned Control Plane scaling tier for the cluster.
</ResponseField>
