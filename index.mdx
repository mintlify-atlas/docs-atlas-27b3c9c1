## What is terraform-aws-eks?

`terraform-aws-eks` is a community-maintained Terraform module that creates production-ready Amazon EKS clusters. It wraps the AWS EKS service with opinionated defaults and exposes a rich set of configuration options — so you get a secure, well-configured cluster without writing hundreds of lines of Terraform from scratch.

The module handles the full lifecycle of an EKS cluster:

- **Control plane** — EKS cluster, IAM roles, KMS encryption, CloudWatch logging, security groups
- **Data plane** — EKS managed node groups, self-managed node groups, Fargate profiles, EKS Auto Mode
- **Access** — Cluster access entries, IRSA (IAM Roles for Service Accounts), OIDC provider
- **Add-ons** — CoreDNS, kube-proxy, VPC CNI, EKS Pod Identity Agent, and any community add-on

```hcl terraform-aws-eks usage
module "eks" {
  source  = "terraform-aws-modules/eks/aws"
  version = "~> 21.0"

  name               = "my-cluster"
  kubernetes_version = "1.33"

  vpc_id     = "vpc-1234556abcdef"
  subnet_ids = ["subnet-abcde012", "subnet-bcde012a", "subnet-fghi345a"]

  # EKS Auto Mode — fully managed data plane
  compute_config = {
    enabled    = true
    node_pools = ["general-purpose"]
  }

  enable_cluster_creator_admin_permissions = true

  tags = {
    Environment = "production"
    Terraform   = "true"
  }
}
```

## Compute options at a glance

<Tabs>
  <Tab title="EKS Auto Mode">
    AWS manages the entire data plane. You define node pool types; AWS handles provisioning, scaling, patching, and termination.

    ```hcl
    compute_config = {
      enabled    = true
      node_pools = ["general-purpose", "system"]
    }
    ```

    Best for: Teams that want minimal operational overhead and are comfortable with AWS-managed nodes.
  </Tab>
  <Tab title="Managed Node Groups">
    AWS manages the EC2 instances within the node group, but you control instance types, AMIs, and scaling parameters.

    ```hcl
    eks_managed_node_groups = {
      main = {
        ami_type       = "AL2023_x86_64_STANDARD"
        instance_types = ["m5.xlarge"]
        min_size       = 2
        max_size       = 10
        desired_size   = 2
      }
    }
    ```

    Best for: Most production workloads needing control over instance types and AMIs.
  </Tab>
  <Tab title="Self-Managed Node Groups">
    You own the EC2 Auto Scaling group. Full control over every launch template parameter.

    ```hcl
    self_managed_node_groups = {
      main = {
        instance_type = "m5.xlarge"
        min_size      = 2
        max_size      = 10
        desired_size  = 2
      }
    }
    ```

    Best for: Advanced use cases requiring mixed instance policies, custom lifecycle hooks, or placement groups.
  </Tab>
  <Tab title="Fargate">
    Serverless — each pod runs in its own isolated compute environment. No nodes to manage.

    ```hcl
    fargate_profiles = {
      default = {
        selectors = [
          { namespace = "kube-system" },
          { namespace = "default" }
        ]
      }
    }
    ```

    Best for: Batch workloads, dev/test environments, or workloads that don't need persistent local storage.
  </Tab>
</Tabs>

## Getting started

<Steps>
  <Step title="Add the module to your Terraform configuration">
    Reference the module from the Terraform Registry and pin to a version:

    ```hcl
    module "eks" {
      source  = "terraform-aws-modules/eks/aws"
      version = "~> 21.0"
    }
    ```
  </Step>
  <Step title="Configure your cluster">
    Set the required variables — cluster name, Kubernetes version, VPC, and subnets. Choose your compute strategy.
  </Step>
  <Step title="Initialize and apply">
    ```bash
    terraform init
    terraform plan
    terraform apply
    ```
  </Step>
  <Step title="Access your cluster">
    ```bash
    aws eks update-kubeconfig --name my-cluster --region us-east-1
    kubectl get nodes
    ```
  </Step>
</Steps>
